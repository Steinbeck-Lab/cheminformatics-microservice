**Part C: isDarkMode Ternary Cleanup**

3. Sweep ALL pages and components for `isDarkMode` ternary patterns in className:
   ```typescript
   // BEFORE:
   className={isDarkMode ? "bg-slate-800 text-white" : "bg-white text-slate-900"}
   // AFTER:
   className="bg-card text-card-foreground"
   ```

   Map common patterns:
   - `isDarkMode ? "bg-slate-800" : "bg-white"` -> `bg-card`
   - `isDarkMode ? "text-white" : "text-slate-900"` -> `text-foreground`
   - `isDarkMode ? "text-slate-300" : "text-slate-600"` -> `text-muted-foreground`
   - `isDarkMode ? "border-slate-700" : "border-slate-300"` -> `border-border`
   - `isDarkMode ? "bg-gray-950" : "bg-slate-100"` -> `bg-background`
   - `isDarkMode ? "text-sky-400" : "text-sky-600"` -> `text-primary`
   - `isDarkMode ? "bg-slate-700" : "bg-slate-200"` -> `bg-secondary`
   - `isDarkMode ? "bg-slate-900/50" : "bg-slate-50"` -> `bg-muted`

   This eliminates the need to import useAppContext just for isDarkMode in styling-only usage. The CSS variables handle the light/dark switch automatically.

   NOTE: Some isDarkMode uses may be legitimate (e.g., passing different props to non-CSS APIs, different SVG fills, or image sources). Only replace className ternaries, not logic ternaries.

4. For the 4 tabbed pages (ChemPage, DepictPage, ConvertPage, ToolsPage):
   - Update tab button styling to use shadcn CSS variables
   - Keep the LayoutGroup tab indicator animation
   - Replace isDarkMode ternaries in tab styling with CSS variable equivalents
   - Example: tab indicator background `isDarkMode ? "bg-sky-400" : "bg-sky-600"` -> `bg-primary`
