1. Add import: `import { Button } from "@/components/ui/button";`
2. Add import if needed: `import { cn } from "@/lib/utils";`

3. Map existing button styles to Button variants:
   - Buttons with blue/sky/primary background colors -> `variant="default"`
   - Buttons with gray/slate/secondary colors -> `variant="secondary"`
   - Buttons with outline/border-only styling -> `variant="outline"`
   - Buttons with red/danger colors -> `variant="destructive"`
   - Buttons with no background (icon-only or minimal) -> `variant="ghost"`
   - Icon-only buttons -> `variant="ghost" size="icon"`
   - Text-link style buttons -> `variant="link"`

4. Map existing button sizes:
   - Small buttons (text-sm, px-2, py-1) -> `size="sm"`
   - Default/medium buttons -> `size="default"` (or omit)
   - Large buttons (text-lg, px-6, py-3) -> `size="lg"`

5. Preserve all existing props: onClick, disabled, type, aria-label, etc.

6. For buttons inside motion.div wrappers (whileHover, whileTap, etc.), keep the motion wrapper and put the Button inside:
   ```typescript
   // BEFORE:
   <motion.button whileHover={{ scale: 1.05 }} className="bg-sky-600 ...">
     Submit
   </motion.button>
   // AFTER:
   <motion.div whileHover={{ scale: 1.05 }} whileTap={{ scale: 0.98 }}>
     <Button variant="default">Submit</Button>
   </motion.div>
   ```
   If motion props are on the button itself (motion.button), convert to motion.div wrapper with Button inside.

7. For buttons with custom classes beyond what Button variant provides, use cn():
   ```typescript
   <Button variant="default" className={cn("w-full mt-4")}>Submit</Button>
   ```

   - Toggle/switch buttons: use `variant="ghost"` or `variant="outline"` with conditional className

IMPORTANT: Do NOT replace tab buttons that are part of LayoutGroup animated tab indicators. These use custom styling for the active/inactive animation and should NOT be converted to shadcn Button.

**Part B: Form Input Migration**

1. Replace all `<input` elements with shadcn `<Input`:
   ```typescript
   import { Input } from "@/components/ui/input";
   // BEFORE:
   <input type="text" className="..." value={val} onChange={handleChange} />
   // AFTER:
   <Input type="text" value={val} onChange={handleChange} />
   ```
   The Input component handles its own styling. Remove inline Tailwind classes that shadcn Input already handles (border, rounded, padding, focus ring). Keep layout classes (w-full, mt-2, etc.).

2. Replace `<select` elements with shadcn `<Select`:
   ```typescript
   import { Select, SelectContent, SelectItem, SelectTrigger, SelectValue } from "@/components/ui/select";
   // BEFORE:
   <select value={val} onChange={(e) => setVal(e.target.value)}>
     <option value="a">A</option>
   </select>
   // AFTER:
   <Select value={val} onValueChange={setVal}>
     <SelectTrigger>
       <SelectValue placeholder="Select..." />
     </SelectTrigger>
     <SelectContent>
       <SelectItem value="a">A</SelectItem>
     </SelectContent>
   </Select>
   ```
   IMPORTANT: shadcn Select uses `onValueChange` (direct value, not event) instead of `onChange` (event). Update handler functions accordingly.

3. Replace `<textarea` elements with shadcn `<Textarea`:
   ```typescript
   import { Textarea } from "@/components/ui/textarea";
   // BEFORE:
   <textarea className="..." value={val} onChange={handleChange} rows={4} />
   // AFTER:
   <Textarea value={val} onChange={handleChange} rows={4} />
   ```

4. For file inputs (`<input type="file">`): Keep as native HTML -- shadcn/ui does not have a file input component. Style it to match by using cn() with appropriate classes.
