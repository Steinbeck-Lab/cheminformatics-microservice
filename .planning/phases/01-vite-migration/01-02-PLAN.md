  **Step 2: Create frontend/eslint.config.mjs**

  Use the exact pattern from 01-RESEARCH.md Pattern 3 (ESLint v9 Flat Config). The file must:
  - Use `.mjs` extension (forces ESM without `"type": "module"` in package.json)
  - Import from `@eslint/js` (comes bundled with eslint 9.x), `globals`, `eslint-plugin-react`, `eslint-plugin-react-hooks`, `eslint-plugin-jsx-a11y`, `eslint-config-prettier`
  - Include `js.configs.recommended` as first element
  - Configure files pattern: `['**/*.{js,jsx}']`
  - Set `languageOptions.globals` to include `globals.browser` and `globals.es2021`
  - Set `settings.react.version` to `'detect'`
  - Spread `reactPlugin.configs.recommended.rules` and `reactPlugin.configs['jsx-runtime'].rules`
  - Add `react-hooks/rules-of-hooks: 'error'` and `react-hooks/exhaustive-deps: 'warn'`
  - Spread `jsxA11y.configs.recommended.rules`
  - Include `prettierConfig` as last rule set (disables formatting rules)
  - Add ignores: `['dist/', 'build/', 'node_modules/', 'public/standalone/']`

  Note: The existing codebase may have ESLint issues (unused variables, missing prop types, etc.). The executor should run `npm run lint` after creation and assess. If there are many errors, add specific rule overrides to reduce noise (e.g., `'react/prop-types': 'off'` since the project doesn't use PropTypes, `'no-unused-vars': 'warn'`). The goal is a working lint config, not zero warnings on day one.

  **Step 2: Create .github/workflows/frontend-test.yml**

  Create the CI workflow using the exact pattern from 01-RESEARCH.md CI Workflow Example:

  ```yaml
  name: frontend-test

  on:
    pull_request:
      branches: [main, development]
      paths:
        - 'frontend/**'
    push:
      branches: [main, development]
      paths:
        - 'frontend/**'

  jobs:
    test:
      runs-on: ubuntu-latest
      defaults:
        run:
          working-directory: frontend
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with:
            node-version: '22'
            cache: 'npm'
            cache-dependency-path: frontend/package-lock.json
        - run: npm ci
        - name: ESLint
          run: npm run lint
        - name: Prettier
          run: npm run format:check
        - name: Tests
          run: npm run test
        - name: Security audit
          run: npm audit --audit-level=low
        - name: Production build
          run: npm run build
  ```

  Per user decision: This CI is a separate workflow from the backend test.yml. It only triggers on changes under `frontend/`.
