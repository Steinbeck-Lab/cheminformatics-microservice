**Step 3: Create `ResizableToolPanel.tsx`:**
```typescript
interface ResizableToolPanelProps {
  toolId: string;           // Used for localStorage key
  inputContent: React.ReactNode;  // Left/top panel content
  outputContent: React.ReactNode; // Right/bottom panel content
  defaultInputSize?: number; // Default left panel % (default 45)
  defaultOutputSize?: number; // Default right panel % (default 55)
  minSize?: number;          // Minimum panel % (default 25)
}
```

Implementation:
- Use `useMediaQuery('(min-width: 1024px)')` to detect wide screen.
- **Wide screen (>1024px):** Render `ResizablePanelGroup` with `direction="horizontal"` and `autoSaveId={`panel-${toolId}`}` for localStorage persistence. Left `ResizablePanel` gets inputContent, `ResizableHandle withHandle` in middle, right `ResizablePanel` gets outputContent. Both panels have `overflow-auto` and padding.
- **Narrow screen (<1024px):** Render a simple stacked layout: `<div className="space-y-4">{inputContent}{outputContent}</div>`. No resizable panels on mobile.
- The `autoSaveId` prop on `ResizablePanelGroup` handles localStorage persistence automatically (per research -- do NOT hand-roll localStorage management).
- Style the ResizableHandle divider: glass-bold appearance with a subtle grip indicator.

Per RESEARCH.md Pattern 4, this is straightforward. The key is using autoSaveId for persistence.
