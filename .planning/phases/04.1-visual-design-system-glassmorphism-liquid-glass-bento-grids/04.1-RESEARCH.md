# Phase 04.1: Visual Design System -- Glassmorphism, Liquid Glass & Bento Grids - Research

**Researched:** 2026-03-13
**Domain:** CSS visual effects (glassmorphism, claymorphism, liquid glass), layout systems (bento grids), animation (motion/react layout), resizable panels, comparison UI
**Confidence:** HIGH

## Summary

This phase transforms the existing frontend from a functional tab-based UI into a cohesive glassmorphism + liquid glass fusion aesthetic across all 9 pages. The core technical challenges are: (1) upgrading the existing `glass` CSS utility to bold glassmorphism with per-page gradient mesh backgrounds, (2) replacing tab-based navigation on 4 pages with animated bento grid layouts using motion/react `layoutId`, (3) implementing resizable split panels using shadcn/ui's Resizable component (built on react-resizable-panels), (4) adding a comparison tray with side-by-side molecule view, and (5) blending claymorphism accents onto interactive elements.

The existing codebase already uses motion/react v12.36.0 with spring physics, shadcn/ui components, Tailwind v4 with CSS-first config and OKLCH variables, and a class-based dark mode system. This foundation is solid. The main risk areas are: SVG filter performance for liquid glass refraction on lower-end devices (mitigated by desktop-only restriction), the complexity of the bento grid expand/collapse interaction pattern (mitigated by motion/react's battle-tested layoutId), and ensuring glass effects look correct in both light and dark modes with consistent intensity.

**Primary recommendation:** Build a layered CSS utility system in tailwind.css (glass-bold, clay-button, gradient-mesh-*) first, then implement new layout components (BentoGrid, ResizablePanel wrapper, ComparisonTray) as reusable primitives, then apply them page-by-page.

### Pattern 1: Glass Utility System (CSS-first approach)
**What:** Define all glass, clay, and gradient utilities as Tailwind v4 @utility directives in tailwind.css
**When to use:** Always -- this is the foundation layer applied via className strings
**Example:**
```css

  backdrop-filter: blur(24px);
  -webkit-backdrop-filter: blur(24px);
  border: 1px solid rgb(255 255 255 / 0.3);
  box-shadow: inset 0 1px 1px 0 rgb(255 255 255 / 0.15);

  &:where(.dark, .dark *) {
    background-color: rgb(30 41 59 / 0.2);
    border-color: rgb(148 163 184 / 0.15);
    box-shadow: inset 0 1px 1px 0 rgb(255 255 255 / 0.05);
```

### Pattern 3: Bento Grid with Expand-in-Place
**What:** CSS Grid layout with motion/react layoutId for smooth expand/collapse
**When to use:** ChemPage, ConvertPage, DepictPage, ToolsPage, HomePage features
**Example:**
```tsx
import { motion, AnimatePresence, LayoutGroup } from "motion/react";

interface BentoCardProps {
  tool: ToolDefinition;
  isExpanded: boolean;
  onToggle: () => void;
  isOtherExpanded: boolean; // Another card is expanded
}

const BentoCard = ({ tool, isExpanded, onToggle, isOtherExpanded }: BentoCardProps) => (
  <motion.div
    layoutId={`bento-card-${tool.id}`}
    className={cn(
      "glass-bold rounded-2xl cursor-pointer overflow-hidden",
      isExpanded && "col-span-full row-span-full z-10",
      isOtherExpanded && !isExpanded && "opacity-30 scale-95 pointer-events-none"
    )}
    onClick={!isExpanded ? onToggle : undefined}
    transition={{ type: "spring", stiffness: 300, damping: 25 }}
    style={{ originX: 0.5, originY: 0 }}
  >
    {/* Card header -- always visible */}
    <motion.div className="p-6 flex items-center gap-4" layout="position">
      <tool.icon className="h-6 w-6" />
      <div>
        <h3 className="font-semibold text-lg">{tool.name}</h3>
        <p className="text-sm text-muted-foreground">{tool.description}</p>
      </div>
    </motion.div>

    {/* Expanded content -- tool UI */}
    <AnimatePresence>
      {isExpanded && (
        <motion.div
          initial={{ opacity: 0 }}
          animate={{ opacity: 1, transition: { delay: 0.2 } }}
          exit={{ opacity: 0 }}
          className="p-6 pt-0"
        >
          {/* Close button */}
          <Button onClick={onToggle} variant="ghost" size="icon" className="absolute top-4 right-4">
            <X className="h-5 w-5" />
          </Button>
          {/* Resizable tool panel */}
          <ResizableToolPanel toolId={tool.id}>
            <tool.component />
          </ResizableToolPanel>
        </motion.div>
      )}
    </AnimatePresence>
  </motion.div>
);

// Page usage
const ChemPage = () => {
  const [expandedToolId, setExpandedToolId] = useState<string | null>(null);

  return (
    <LayoutGroup>
      <div className={cn(
        "grid gap-4",
        // Responsive bento sizing
        "grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4",
        expandedToolId && "grid-cols-1" // Collapse to single column when expanded
      )}>
        {tools.map(tool => (
          <BentoCard
            key={tool.id}
            tool={tool}
            isExpanded={expandedToolId === tool.id}
            onToggle={() => setExpandedToolId(prev => prev === tool.id ? null : tool.id)}
            isOtherExpanded={expandedToolId !== null && expandedToolId !== tool.id}
          />
        ))}
      </div>
    </LayoutGroup>
  );
};
```

### Pattern 4: Resizable Tool Panel with Persistence
**What:** shadcn/ui Resizable wrapping input/output with localStorage save
**When to use:** Inside expanded bento cards on wide screens (>1024px)
**Example:**
```tsx
import { ResizablePanelGroup, ResizablePanel, ResizableHandle } from "@/components/ui/resizable";

const ResizableToolPanel = ({ toolId, children }: { toolId: string; children: React.ReactNode }) => {
  const storageKey = `panel-${toolId}`;
  const isWideScreen = useMediaQuery("(min-width: 1024px)");

  // On narrow screens, render stacked (no resize)
  if (!isWideScreen) {
    return <div className="space-y-4">{children}</div>;
  }

  return (
    <ResizablePanelGroup
      direction="horizontal"
      autoSaveId={storageKey}
      className="min-h-[400px] rounded-lg"
    >
      <ResizablePanel defaultSize={45} minSize={25}>
        {/* Input panel -- rendered by tool component's input section */}
        <div className="h-full p-4 overflow-auto">{/* Input UI */}</div>
      </ResizablePanel>
      <ResizableHandle withHandle className="mx-1" />
      <ResizablePanel defaultSize={55} minSize={25}>
        {/* Output panel -- rendered by tool component's output section */}
        <div className="h-full p-4 overflow-auto">{/* Output UI */}</div>
      </ResizablePanel>
    </ResizablePanelGroup>
  );
};
```

Note: `autoSaveId` is a built-in feature of react-resizable-panels that handles localStorage persistence automatically.

### Pattern 5: Liquid Glass SVG Refraction (Desktop Only)
**What:** SVG filter using feTurbulence + feDisplacementMap for hover-activated refraction
**When to use:** Applied as a CSS filter on glass panels on hover, desktop only
**Example:**
```tsx
// Hidden SVG filter definition (place once in layout or App.tsx)
<svg width="0" height="0" style={{ position: "absolute" }}>
  <defs>
    <filter id="liquid-glass-refraction" x="-10%" y="-10%" width="120%" height="120%">
      <feTurbulence
        type="fractalNoise"
        baseFrequency="0.015"
        numOctaves="3"
        seed="2"
        result="turbulence"
      />
      <feDisplacementMap
        in="SourceGraphic"
        in2="turbulence"
        scale="6"
        xChannelSelector="R"
        yChannelSelector="G"
      />
    </filter>
  </defs>
</svg>

// CSS approach for hover activation
// In tailwind.css or component styles:
.glass-refraction-hover:hover {
  filter: url(#liquid-glass-refraction);
  transform: scale(1.005);
  transition: filter 300ms ease, transform 300ms ease;
}

// Mobile exclusion via media query
@media (hover: none) {
  .glass-refraction-hover:hover {
    filter: none;
    transform: none;
  }
}
```

**Recommended SVG filter parameters (Claude's discretion):**
- `baseFrequency`: 0.012-0.018 (lower = larger, gentler distortion)
- `numOctaves`: 2-3 (higher = more detail, more GPU cost)
- `scale`: 4-8 (displacement intensity; higher = more warp)
- `seed`: any integer (determines distortion pattern)

**Performance note:** feDisplacementMap is slower than feGaussianBlur. The desktop-only restriction is critical. Limit to hover state only (not always-on).

### Anti-Patterns to Avoid
- **Nesting glass layers:** Never apply backdrop-filter on an element inside another backdrop-filter element. It multiplies GPU cost and can cause visual artifacts. Use transparent backgrounds for inner elements instead.
- **Animating blur values:** Never transition or animate the backdrop-filter blur amount. Use opacity transitions on the glass element itself, or animate only transform/opacity.
- **Too many glass elements per viewport:** Keep to 3-5 glass elements visible simultaneously. The bento grid naturally limits this when cards are collapsed.
- **Global CSS filter on body:** Do not apply SVG filters globally. Apply only to specific hover targets.
- **Building custom drag resize:** Use react-resizable-panels' autoSaveId for persistence instead of manual localStorage management.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Resizable panels | Custom mousedown/mousemove drag handler | shadcn/ui Resizable (react-resizable-panels) | Keyboard access, ARIA, edge snapping, min/max constraints, autoSaveId persistence |
| Layout persistence | Manual localStorage read/write for panels | autoSaveId prop on ResizablePanelGroup | Built-in feature, handles serialization, debouncing |
| Expand/collapse animation | CSS transition on height:auto | motion/react layoutId + LayoutGroup | height:auto cannot be transitioned in CSS; layoutId handles FLIP animation automatically |
| Responsive breakpoints | window.innerWidth in useEffect | CSS media queries + optional useMediaQuery hook | CSS handles most cases; hook only for conditional rendering |
| Gradient mesh animation | Canvas-based animated gradient | CSS radial-gradient with static composition | CSS is simpler, no JS overhead, sufficient for the aesthetic |
| Comparison state | Redux or prop drilling | React Context (ComparisonContext) | Max 2 molecules, minimal state, no need for external state library |

**Key insight:** The existing stack (motion/react, shadcn/ui, Tailwind v4) already provides nearly everything needed. The only new dependency is react-resizable-panels (via shadcn Resizable CLI install). All visual effects are achievable with CSS utilities + SVG filters + motion/react animation primitives.

## Common Pitfalls

### Pitfall 1: Backdrop-filter Performance on Mobile
**What goes wrong:** blur(24px) backdrop-filter causes janky scrolling and high GPU usage on mid-range mobile devices
**Why it happens:** Each glass element triggers a separate GPU blur pass on every frame during scroll
**How to avoid:** Use the decided mobile strategy: reduce to blur(12px) on <768px viewports. Limit glass elements to 3-5 per viewport. Never animate elements that have backdrop-filter.
**Warning signs:** Lighthouse Performance score drops below 60 on mobile; visible scroll jank on Android mid-range

### Pitfall 2: SVG Filter Browser Compatibility
**What goes wrong:** SVG filters applied via CSS `filter: url(#id)` may not work consistently as backdrop-filter in non-Chromium browsers (Safari, Firefox)
**Why it happens:** SVG filter support varies; using SVG filters as backdrop-filter value only works reliably in Chromium
**How to avoid:** Apply SVG refraction filter as a regular `filter` property on the element (not backdrop-filter). Use `@supports` or feature detection. Keep it hover-only so the at-rest state is always clean.
**Warning signs:** Refraction effect missing in Safari; test in Safari during development

### Pitfall 3: LayoutGroup Scope Conflicts
**What goes wrong:** motion/react layoutId animations jump across unrelated components, causing visual glitches
**Why it happens:** layoutId is global within a LayoutGroup; if two LayoutGroups share IDs, animations cross-pollinate
**How to avoid:** Use unique layoutId prefixes per page/section (e.g., `bento-chem-{tool.id}`, `bento-depict-{tool.id}`). Each page's bento grid should have its own LayoutGroup with a unique `id` prop.
**Warning signs:** Cards from one page animate toward cards on a different page during route transitions

### Pitfall 4: Glass Intensity Inconsistency Between Modes
**What goes wrong:** Glass panels look great in dark mode but nearly invisible (or too opaque) in light mode, or vice versa
**Why it happens:** The same white/alpha value reads very differently against light vs dark backgrounds
**How to avoid:** The user locked "same glass intensity in both modes, shift color palette only." Test both modes at every step. Use the exact decided values: bg white/10 light, slate-800/20 dark, border white/30 both.
**Warning signs:** Light mode glass panels look flat/washed out or too opaque

### Pitfall 5: Bento Grid Expand Breaks Page Scroll
**What goes wrong:** When a bento card expands to fill the grid area, the page height changes drastically, causing the viewport to jump
**Why it happens:** layout animation moves elements but does not manage scroll position
**How to avoid:** When expanding, scroll the expanded card into view with `scrollIntoView({ behavior: 'smooth', block: 'start' })`. Consider using `position: sticky` or ensure the expanded card's top aligns with where the user clicked.
**Warning signs:** Users lose their scroll position after expanding/collapsing a card

### Pitfall 6: react-resizable-panels v4 API Changes
**What goes wrong:** Using old API from react-resizable-panels v2/v3 with the v4 package
**Why it happens:** shadcn/ui recently updated to v4; some documentation still shows old patterns
**How to avoid:** Use shadcn CLI to generate the wrapper (handles API mapping). The wrapper exports ResizablePanelGroup, ResizablePanel, ResizableHandle with stable props. Use `orientation` not `direction` if going to the raw library.
**Warning signs:** Console warnings about deprecated props; data-panel-group-direction not working

### Pitfall 7: Claymorphism in Dark Mode
**What goes wrong:** Clay shadow effects (especially the light inner shadow) look unnatural on dark backgrounds
**Why it happens:** Claymorphism was designed for bright pastel backgrounds; dark mode inverts the lighting model
**How to avoid:** Reduce inner light shadow intensity in dark mode (white/8% instead of white/40%). Increase outer shadow darkness. The clay utility should have explicit dark mode variants.
**Warning signs:** Buttons look like they have a white line on top in dark mode

### Example 2: useMediaQuery Hook
```tsx
import { useState, useEffect } from "react";

export function useMediaQuery(query: string): boolean {
  const [matches, setMatches] = useState(false);

  useEffect(() => {
    const mql = window.matchMedia(query);
    setMatches(mql.matches);
    const handler = (e: MediaQueryListEvent) => setMatches(e.matches);
    mql.addEventListener("change", handler);
    return () => mql.removeEventListener("change", handler);
  }, [query]);

  return matches;
}
```

### Example 3: Comparison Context
```tsx
interface ComparisonMolecule {
  smiles: string;
  title: string;
  imageUrl?: string;
  descriptors?: Record<string, unknown>;
}

interface ComparisonContextValue {
  molecules: ComparisonMolecule[];
  isOpen: boolean;
  canAdd: boolean; // molecules.length < 2
  addMolecule: (mol: ComparisonMolecule) => void;
  removeMolecule: (smiles: string) => void;
  clearAll: () => void;
  openComparison: () => void;
  closeComparison: () => void;
}

const ComparisonContext = createContext<ComparisonContextValue | undefined>(undefined);

export const ComparisonProvider = ({ children }: { children: React.ReactNode }) => {
  const [molecules, setMolecules] = useState<ComparisonMolecule[]>([]);
  const [isOpen, setIsOpen] = useState(false);

  const addMolecule = (mol: ComparisonMolecule) => {
    setMolecules(prev => {
      if (prev.length >= 2) return prev; // Max 2
      if (prev.some(m => m.smiles === mol.smiles)) return prev; // No duplicates
      const updated = [...prev, mol];
      if (updated.length === 1) setIsOpen(true); // Auto-open tray on first add
      return updated;
    });
  };

  // ... removeMolecule, clearAll, etc.

  return (
    <ComparisonContext.Provider value={{
      molecules, isOpen, canAdd: molecules.length < 2,
      addMolecule, removeMolecule, clearAll,
      openComparison: () => setIsOpen(true),
      closeComparison: () => setIsOpen(false),
    }}>
      {children}
    </ComparisonContext.Provider>
  );
};
```

## Open Questions

1. **SVG filter Safari rendering**
   - What we know: SVG feTurbulence+feDisplacementMap works as CSS `filter` in all browsers, but as `backdrop-filter` value only in Chromium
   - What's unclear: Whether applying as regular `filter` on a positioned overlay element looks the same as backdrop-filter approach
   - Recommendation: Implement as regular `filter` on the glass element hover state (not backdrop-filter). Test in Safari early. If visual quality is insufficient, fall back to a simpler CSS-only distortion (e.g., slight scale + brightness shift) for non-Chromium browsers.

2. **Bento expand animation scroll management**
   - What we know: motion/react handles the FLIP animation, but does not auto-scroll
   - What's unclear: Best UX for long pages -- should expanded card scroll to top, or should viewport maintain relative position?
   - Recommendation: Use `scrollIntoView({ behavior: 'smooth', block: 'nearest' })` after expansion animation completes. Test with different page lengths.

3. **Tool component input/output split for resizable panels**
   - What we know: Current tool components (e.g., StereoisomersView, DescriptorsView) render input and output as a single component
   - What's unclear: Whether tools need refactoring to expose input/output as separate render sections, or if the resizable panel wraps the entire tool with a CSS-based split
   - Recommendation: Keep existing tool components intact for now. The resizable panel should wrap the entire tool component, with the tool itself deciding its internal layout. If specific tools need input/output split, they can be refactored individually in later iterations. For this phase, the resizable panel is the container for the expanded bento card content.
