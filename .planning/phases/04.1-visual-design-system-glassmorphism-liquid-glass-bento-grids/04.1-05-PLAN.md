**1. `frontend/src/context/ComparisonContext.tsx`:**

Define types:
```typescript
interface ComparisonMolecule {
  smiles: string;
  title: string;        // Tool name or molecule name
  imageUrl?: string;    // Structure depiction URL if available
  descriptors?: Record<string, string | number>; // MW, formula, etc.
  sourceToolId?: string; // Which tool generated this result
}

interface ComparisonContextValue {
  molecules: ComparisonMolecule[];
  isOpen: boolean;          // Is tray visible
  isComparing: boolean;     // Is side-by-side view active
  canAdd: boolean;          // molecules.length < 2
  addMolecule: (mol: ComparisonMolecule) => void;
  removeMolecule: (smiles: string) => void;
  clearAll: () => void;
  openTray: () => void;
  closeTray: () => void;
  startCompare: () => void;  // Switch to side-by-side view
  stopCompare: () => void;   // Back to tray mode
}
```

Implementation per RESEARCH.md Example 3:
- `addMolecule`: Max 2 check, no duplicate SMILES, auto-open tray on first add
- `removeMolecule`: Remove by SMILES, close tray if last molecule removed, stop comparing if below 2
- `clearAll`: Reset to empty state, close tray, stop comparing
- Use React Context (NOT zustand -- per research recommendation for max 2 items)

Export `ComparisonProvider` and `useComparison` hook (with runtime guard like existing AppContext pattern).

**Wire the "Add to Compare" button into the ChemPage BentoCard expanded views:**

1. Import `AddToCompareButton` from `@/components/common/AddToCompareButton`

2. Update the BentoCard rendering in ChemPage to include an AddToCompareButton in the expanded card area. The button should appear in the card header area (next to the close button) or in a toolbar row below the card header when a tool is expanded.

   The simplest approach: Add the AddToCompareButton to BentoCard's expanded header row. Since BentoCard accepts children (the tool component), and the tool component manages its own SMILES input state, the AddToCompareButton needs access to the current SMILES being analyzed.

   **Practical approach:** Add the AddToCompareButton to the BentoCard expanded header for tools that produce molecule results. Pass a callback or state through the BentoCard:

   ```tsx
   {tabs.map(tab => (
     <BentoCard
       key={tab.id}
       id={tab.id}
       pageKey="chem"
       name={tab.name}
       description={tab.description}
       icon={tab.icon}
       size={getBentoSize('chem', tab.id)}
       isExpanded={expandedToolId === tab.id}
       isOtherExpanded={expandedToolId !== null && expandedToolId !== tab.id}
       onToggle={...}
       headerExtra={
         expandedToolId === tab.id ? (
           <AddToCompareButton
             smiles={currentSmiles || ''}
             title={tab.name}
             sourceToolId={tab.id}
           />
         ) : undefined
       }
     >
       <tab.component />
     </BentoCard>
   ))}
   ```

   This requires adding a `headerExtra` prop to BentoCardProps in BentoGrid.tsx:
   ```typescript
   interface BentoCardProps {
     // ... existing props
     headerExtra?: React.ReactNode; // Extra content rendered in expanded card header (e.g., AddToCompareButton)
   }
   ```

   Update BentoGrid.tsx's BentoCard to render `headerExtra` in the expanded header row, next to the close button.

3. For the SMILES state: ChemPage already has the SMILES input for its tools. The simplest approach is to maintain a `currentSmiles` state at the ChemPage level that gets updated when users enter SMILES in any tool. If this is too complex, start with the AddToCompareButton having an empty smiles string and rely on users to have entered a SMILES -- the button simply captures whatever is in the current context.

   A pragmatic approach: Add `currentSmiles` state to ChemPage, pass it through existing tool components if they expose an onChange, or keep it simple and let the AddToCompareButton prompt the user for SMILES if not available.

4. The key outcome: When a user expands a tool card on ChemPage, they see an "Add to Compare" button in the card header that they can click to add the current molecule to the comparison tray.
