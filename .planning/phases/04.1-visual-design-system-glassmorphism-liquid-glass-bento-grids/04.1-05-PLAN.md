**1. `frontend/src/context/ComparisonContext.tsx`:**

Define types:
```typescript
interface ComparisonMolecule {
  smiles: string;
  title: string;        // Tool name or molecule name
  imageUrl?: string;    // Structure depiction URL if available
  descriptors?: Record<string, string | number>; // MW, formula, etc.
  sourceToolId?: string; // Which tool generated this result
}

interface ComparisonContextValue {
  molecules: ComparisonMolecule[];
  isOpen: boolean;          // Is tray visible
  isComparing: boolean;     // Is side-by-side view active
  canAdd: boolean;          // molecules.length < 2
  addMolecule: (mol: ComparisonMolecule) => void;
  removeMolecule: (smiles: string) => void;
  clearAll: () => void;
  openTray: () => void;
  closeTray: () => void;
  startCompare: () => void;  // Switch to side-by-side view
  stopCompare: () => void;   // Back to tray mode
}
```

Implementation per RESEARCH.md Example 3:
- `addMolecule`: Max 2 check, no duplicate SMILES, auto-open tray on first add
- `removeMolecule`: Remove by SMILES, close tray if last molecule removed, stop comparing if below 2
- `clearAll`: Reset to empty state, close tray, stop comparing
- Use React Context (NOT zustand -- per research recommendation for max 2 items)

Export `ComparisonProvider` and `useComparison` hook (with runtime guard like existing AppContext pattern).

**2. `frontend/src/components/common/ComparisonTray.tsx`:**

The tray is a fixed-position bar at the bottom of the viewport.

Implementation:
- Use `useComparison()` to read state
- `motion.div` with `fixed bottom-0 left-0 right-0 z-50 glass-bold rounded-t-2xl`
- Slide-up animation: `initial={{ y: "100%" }} animate={{ y: isOpen ? 0 : "100%" }}` with spring `{ stiffness: 300, damping: 25 }`
- When open, show:
  - Title: "Compare Molecules" or molecule count
  - For each molecule: small card with title, SMILES (truncated), remove (X) button
  - "Compare Side-by-Side" button (enabled when 2 molecules added), uses `clay-interactive` styling
  - "Clear All" link
- Tray height: auto-sized based on content, max-height constrained
- On mobile: same layout but full-width, slightly more compact

The tray component should be rendered once in App.tsx (outside of routes, after ComparisonProvider wrapping). It will be visible whenever isOpen is true, regardless of current route.
